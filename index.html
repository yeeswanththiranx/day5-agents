Topic 01 of 06
Chatbot vs AI Agent
They look similar. They are fundamentally different.
The Core Difference

A chatbot answers. An agent acts.

This is the single most important distinction in modern AI. Understanding it is the foundation of everything else in Days 6–11.

💬 CHATBOT
You: "Book me a flight to Delhi on Friday."

ChatGPT: "Sure! Here are some steps to book a flight to Delhi on Friday: 1. Go to MakeMyTrip or GoAir. 2. Enter your origin city. 3. Select Friday as departure..."

It told you how. It did nothing.
Responds to questions. Cannot take action. You do all the work.
🤖 AI AGENT
You: "Book me the cheapest flight to Delhi on Friday under ₹5000."

Agent: Opens browser → searches MakeMyTrip → compares prices → finds IndiGo at ₹4,200 → fills booking form → confirms → sends you the ticket by email.

You gave a goal. It completed it.
Takes real actions. Uses tools. Works autonomously toward your goal.
The Office Assistant Analogy
"A chatbot is like a very knowledgeable colleague who gives you advice. An agent is like a personal assistant who actually does the work — makes calls, sends emails, books meetings — while you focus on something else."
📌 The key word for agents: autonomy. They decide HOW to complete a goal, not just what to say about it.

Topic 02 of 06
How Agents Think
The Perception → Reasoning → Action → Feedback loop that drives every agent.
The Agent Loop

Agents don't just answer. They loop.

Every AI agent — from a simple research bot to a full software engineer — runs the same four-step loop continuously until the goal is complete.

STEP 01
👁️
Perceive
Reads input — your goal, tool results, current state of the world
→
STEP 02
🧠
Reason
Thinks: "What is the next best action to take toward the goal?"
→
STEP 03
⚡
Act
Executes the action — search web, write code, send email, click button
→
STEP 04
🔄
Observe
Reads the result. Was it successful? What changed? Loop begins again.

This loop runs over and over until the agent decides the goal is complete — or it gets stuck and asks the human for help. The key insight: each loop iteration produces new information that informs the next action. The agent is constantly updating its understanding of the world.

# Real example — Agent given goal: "Research the top 3 AI companies and write a summary"

LOOP 1:
  Perceive:  Goal = "research top 3 AI companies"
  Reason:    "I need current data. I should search the web first."
  Act:       web_search("top AI companies 2025")
  Observe:   Got results — OpenAI, Google DeepMind, Anthropic mentioned

LOOP 2:
  Perceive:  Search results about OpenAI
  Reason:    "I need more detail about each. Let me fetch each company page."
  Act:       fetch_webpage("openai.com/about")
  Observe:   Got detailed info about OpenAI

LOOP 3 + 4:
  # Repeat for Google DeepMind and Anthropic

LOOP 5:
  Perceive:  All data collected
  Reason:    "I have enough info. Time to write the summary."
  Act:       write_document("summary.md")
  Observe:   Document created. Goal complete. ✓
💡
Key insight: The agent never needed human intervention for each step. You gave it ONE goal. It figured out all the intermediate steps — searching, reading, writing — on its own. That's what makes agents fundamentally different from chatbots.
Topic 03 of 06
Memory in AI Agents
How agents remember — within a session, across sessions, and in external storage.
Three Types of Memory

Agents need memory to work across time.

A chatbot forgets everything when you close the tab. An agent is designed to remember — but memory in AI agents is more complex than it sounds. There are three distinct types.

⚡
Short-Term Memory
Everything in the current context window. The agent sees the full conversation history and tool results from this session. Cleared when session ends.
"What did the user say 3 messages ago? Current task progress? Last tool result?" — all in short-term memory.
🗃️
Long-Term Memory
Stored in a database outside the model. Persists across sessions. Agent retrieves relevant memories using search when needed. Like a personal diary it can look up.
"This user prefers bullet lists. Their name is Karthik. They work at a startup in Chennai." — retrieved next session.
📁
External Memory
Files, databases, documents the agent reads and writes. Not inside the model — exists in the real world. Agent can update, search, and retrieve at any time.
A spreadsheet, a code repository, a PDF, a CRM database — all external memory the agent can read/write.
The Student Analogy
"Short-term memory = what's on your desk right now. Long-term memory = what you remember from last week's class. External memory = your notebook that you can open, read, and write in anytime."
📌 A powerful agent uses all three — current context + past knowledge + external data — to make the best decisions.
⚠️
Common confusion: A basic LLM like ChatGPT has NO long-term memory by default. Each conversation starts fresh. When you see AI "remembering" you across sessions (like ChatGPT's memory feature), that's an external database storing and retrieving notes — not the model itself remembering.

Tools Agents Use
An LLM + tools = an Agent. Tools are what give agents their power to act.
The Power of Tools

An LLM with tools is completely different.

A plain LLM can only generate text. Give it tools — and it can search the internet, run code, send emails, browse websites, manage files. Tools are what transform a language model into an agent.

# How an agent decides which tool to use — simplified

User goal: "What is the current stock price of TCS and plot a 7-day chart?"

Agent reasoning:
  → Need current price? web_search("TCS stock price today")
  → Need 7-day history? api_call("finance_api", symbol="TCS", days=7)

The formula: Agent = LLM (brain) + Tools (hands) + Memory (memory) + Goal (mission). Remove any one of these and the agent becomes significantly less capable. The LLM reasons; the tools act; memory keeps track; the goal drives everything.

Real World Agent Examples
Agents that exist today — what they do and how they work.
Agents in the Wild

These agents exist right now.

These are not theoretical — they are real, deployed agents being used by thousands of companies and developers today.

👨‍💻
Devin (Cognition AI)
The first AI software engineer. Give it a task: "Build a web scraper for this website." It opens VS Code, writes code, runs tests, debugs errors, and pushes to GitHub — without any human input for each step. Works for hours on complex engineering problems.
🔍
Perplexity AI
A research agent that combines LLM reasoning with real-time web search. Ask it anything current — it searches multiple sources, reads them, synthesises the information, and gives you a cited answer. Not a chatbot — it actively browses to answer.

One of the first fully autonomous agents. Give it a goal like "Research competitors and write a report." It breaks the goal into sub-tasks, executes each one using web search and code tools, evaluates results, and loops until the goal is done.

Anthropic's implementation of persistent agent memory. Claude remembers your preferences, ongoing projects, and past conversations across sessions. Agents built on Claude can retain context over weeks of work — not just one conversation.

No-code agents that connect 6,000+ apps. "When a new lead fills my form, research the company, draft a personalised email, and add to CRM." The agent handles all three steps automatically across separate apps.

A framework for multi-agent systems. Build a team of agents with roles: Researcher, Writer, Editor, Fact-Checker. Each agent does its job, then hands off to the next. Like a real team, but fully automated.
Topic 06 of 06
The ReAct Framework
Reason + Act — the most widely used agent reasoning pattern.
Reason + Act

ReAct = Reasoning before every Action.

ReAct is a prompting framework that makes agents write out their reasoning before every action — and observe the result before reasoning again. It dramatically improves agent reliability.

The name ReAct comes from Reasoning + Acting. The key insight is simple: before the agent uses a tool, it writes out WHY it's choosing that tool. This written reasoning helps it stay on track and makes its decisions transparent and debuggable.

REAL ReAct WALKTHROUGH — Goal: "Find the CEO of Infosys and their educational background"
THOUGHT
I need to find who the current CEO of Infosys is. I don't know if this has changed recently so I should search rather than rely on my training data.
ACTION
web_search("Infosys CEO 2025")
OBSERVE
Results show Salil Parekh has been CEO since 2018. He is mentioned in multiple recent articles.
THOUGHT
Good — I have the CEO's name. Now I need his educational background. I should search specifically for that.
ACTION
web_search("Salil Parekh education background")
OBSERVE
IIT Bombay for undergraduate (Computer Science), Cornell University for MBA. Found on LinkedIn and Infosys website.
THOUGHT
I now have all the information needed. I can give the final answer.
ANSWER
Salil Parekh is the CEO of Infosys (since 2018). He studied Computer Science at IIT Bombay and completed his MBA at Cornell University.
✅
Why ReAct matters: Without the Thought step, agents jump straight to actions — often the wrong ones. Writing out reasoning before acting makes agents more reliable, easier to debug, and more transparent. It's the difference between an agent that works and one that goes in circles.

Q 01 / 15💼 Interview
What is the fundamental difference between a chatbot and an AI agent?
✅ Respond vs Act. This is the core distinction. A chatbot's job ends when it generates text. An agent's job begins after reasoning — it uses tools, executes steps, and continues until the goal is achieved. The word "autonomously" is key — the agent decides the steps, not the user.
Q 02 / 15💼 Interview
What does ReAct stand for in the context of AI agents?
✅ Reasoning + Acting. ReAct is a framework where an agent alternates between writing a Thought (reasoning about what to do next) and taking an Action (using a tool). This Thought → Action → Observation loop makes agents more reliable and their decisions transparent.
Q 03 / 15🎯 Scenario
SCENARIO
Priya asks ChatGPT: "Book me a table at a Chennai restaurant for 7pm tonight." ChatGPT replies with a list of restaurants and instructions on how to book. Priya is frustrated — she wanted it done for her.
Why couldn't ChatGPT complete Priya's request automatically?
✅ No tools = no action. Booking a table requires browsing a restaurant website or calling an API — real-world actions. ChatGPT without tools can only generate text. An AI agent with browser and API tools could actually open Zomato, find the restaurant, select 7pm, and complete the booking.
Q 04 / 15💼 Interview
Name the three types of memory used in AI agents.
✅ Short-term, Long-term, External. Short-term = current context window (this session). Long-term = stored in a database, persists across sessions. External = files, spreadsheets, databases the agent can read and write. A robust agent uses all three together.
Q 05 / 15🎯 Scenario
SCENARIO
An AI agent is given the goal: "Research the top 5 electric cars of 2025." After searching the web, it finds the results but they are from 2023. The agent notices this and searches again with "electric cars 2025 latest."
Which part of the agent loop caused the agent to search again?
✅ The Observe step. After acting (searching), the agent observed the results — noticed they were from 2023, not 2025. This observation fed back into reasoning: "Results are outdated. I need a more specific query." Then it acted again. This loop — act → observe → reason → act again — is what makes agents self-correcting.
Q 06 / 15💼 Interview
What is the formula that best describes an AI agent?
✅ LLM + Tools + Memory + Goal. The LLM is the brain — it reasons and decides. Tools are the hands — they let the agent act in the world. Memory keeps track of progress and context. The Goal drives the entire process. Remove any one and the agent becomes significantly weaker.
Q 07 / 15🎯 Scenario
SCENARIO
Karan used Claude to plan a project last Monday. Today, Tuesday the following week, he opens a new chat and asks Claude: "Continue where we left off on the project." Claude replies: "I don't have any information about a previous project."
Why doesn't Claude remember the previous conversation?
✅ No default long-term memory. Base LLMs start every conversation completely blank — no memory of previous sessions. Long-term memory must be explicitly built using external databases and retrieval systems. Features like ChatGPT Memory or Claude Projects are add-ons that store summaries in a database and inject them back into new conversations.
Q 08 / 15💼 Interview
Which AI agent is known as the "first AI software engineer" that can independently write, test and deploy code?
✅ Devin by Cognition AI. Launched in 2024, Devin was marketed as the first AI software engineer. It can open a development environment, write code, run tests, debug errors, read documentation, and push to GitHub — completing full engineering tasks independently. It went viral because it could complete tasks on Upwork coding platforms.
Q 09 / 15🎯 Scenario
SCENARIO
A company builds an agent with these tools: web_search, send_email, read_spreadsheet. An employee asks the agent: "Summarise our Q3 sales data and email it to the team." The agent successfully reads the spreadsheet, writes a summary, and sends the email — all without human help.
Which tools did the agent use in the correct order?
✅ read_spreadsheet → summarise → send_email. The agent first needs the data (read_spreadsheet), then uses its LLM reasoning to generate a summary (no tool needed — this is internal), then sends it (send_email). Web search was not needed — the data was already in the company's spreadsheet. Agents pick the minimal set of tools needed for each task.
Q 10 / 15💼 Interview
In the ReAct framework, what comes between a Thought and an Observation?
✅ Action. The ReAct cycle is: Thought → Action → Observation → Thought → Action → Observation... The Thought is the reasoning ("I should search for X"). The Action is executing it ("web_search('X')"). The Observation is reading the result. Then the cycle repeats with a new Thought informed by the Observation.
Q 11 / 15🎯 Scenario
SCENARIO
A company creates 3 agents: Agent 1 = Researcher (searches and reads), Agent 2 = Writer (drafts content), Agent 3 = Editor (reviews and improves). Each agent hands its output to the next. Which framework does this describe?
What is this type of agent system called?
✅ Multi-agent system. When multiple specialised agents each handle one role and pass outputs to each other, it's a multi-agent system. CrewAI is the most popular framework for building these. Each agent has a defined role, goal, and tools — together they complete complex tasks that a single agent would struggle with.
Q 12 / 15💼 Interview
What is the key advantage of an agent writing a "Thought" before every action in ReAct?
✅ Transparency and error reduction. When an agent must articulate WHY it's doing something before doing it, two things happen: 1) It's less likely to take random or wrong actions because the reasoning constrains the choice. 2) When something goes wrong, you can read the thought trail and see exactly where the reasoning broke down. It's like showing your work in maths.
Q 13 / 15🎯 Scenario
SCENARIO
Vijay builds an AI agent to handle customer support emails. After 3 days, he notices the agent is giving wrong answers about product pricing — even though the correct prices are in a spreadsheet the agent has access to.
What is the most likely cause of this problem?
✅ Agent not using the right tool. LLMs tend to rely on their training data when they should be using a tool. The agent "knows" some prices from training, so it doesn't always query the spreadsheet. The fix: instruct the agent explicitly — "For any pricing question, ALWAYS use the read_spreadsheet tool. Never rely on your own knowledge for prices."
Q 14 / 15🎯 Scenario⚡ Tricky
SCENARIO
An agent is given: "Send a birthday email to everyone in my contacts whose birthday is today." It reads the contacts file, finds 3 people with today's birthday, writes personalised emails, and sends them. Then it loops again and sends the same emails a second time.
What went wrong in the agent's design?
✅ Missing stopping condition. This is a classic agent design error. The agent looped back after completing the task because it wasn't told "stop after sending once." Every agent needs a clear termination condition — otherwise it runs indefinitely. Fix: add "After sending emails, mark the task as complete and stop."
Q 15 / 15💼 Interview⚡ Tricky
An interviewer asks: "If an AI agent and a human employee are given the same task — research competitors and write a report — what is the agent's biggest limitation compared to the human?"
✅ Lack of genuine judgment. Agents are fast and thorough but credulous — they cannot reliably distinguish a trustworthy industry report from a biased marketing piece, a satirical article from real news, or a manipulated statistic from a genuine one. A human expert brings domain knowledge, intuition, and critical thinking that agents still lack. This is why "human in the loop" design remains important for high-stakes tasks.
